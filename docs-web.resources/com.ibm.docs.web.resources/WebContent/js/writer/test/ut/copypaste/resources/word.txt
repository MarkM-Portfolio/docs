<h1>Background<o:p></o:p></h1>

<p class="MsoNormal">The HCL Connections Cloud is managed, maintained, updated
and serviced by IBM as a true SaaS (Software as a Service) platform. IBM
maintains and monitors its platforms using both internal and external tools to
secure availability and performance awareness for SaaS delivery purposes.<br>
It is important to keep in mind that network performance and service
availability are two different things. <br>
With network performance the whole chain of network infrastructure components
and multiple service providers are involved. This includes IBM Softlayer network
infrastructure, your client internal network(s) as well as telecom providers
and not forgetting potential VPN software, proxy servers, firewalls and
workstations or mobile devices. This document is intended as a guide on how to
see availability and performance awareness using the tools out there today.<o:p></o:p></p>

<h1>network acceleration with akamai<o:p></o:p></h1>

<p class="MsoNormal"><span style="color:windowtext;mso-font-kerning:0pt;
mso-fareast-language:EN-US"><a href="http://www-10.lotus.com/ldd/bhwiki.nsf/xpDocViewer.xsp?lookupName=IBM+Redbooks%3A+Preparing+your+enterprise+for+IBM+Connections+Cloud#action=openDocument&amp;res_title=2.1.1_Network_Accelerator&amp;content=pdcontent"><span style="mso-bidi-font-family:Arial;color:#424282">Network acceleration</span></a></span><span style="mso-bidi-font-family:Arial;color:black;mso-font-kerning:0pt;mso-fareast-language:
EN-US"> </span>to the IBM data centers is an active enhanced solution that benefits your users as it will
improve overall responsiveness of the IBM SaaS services worldwide. For network
acceleration, IBM leverages Akamai (<span style="mso-bidi-font-family:
Arial;color:black;mso-font-kerning:0pt;mso-fareast-language:EN-US"><a href="http://www.akamai.com/"><span style="color:#424282">www.akamai.com</span></a></span>) to assist with the optimizing of network traffic between users
worldwide and the IBM data centers. Akamai accomplishes this by calculating the
most efficient route between the data center and the end user based on the
nature of the content, as well as by caching static content on the network
edge, closest to the end user. Akamai resolves the following HCL Connections
Cloud<span style="mso-bidi-font-family:Arial;color:black;mso-font-kerning:
0pt;mso-fareast-language:EN-US"> <a href="http://www-10.lotus.com/ldd/bhwiki.nsf/dx/Cheat_Sheetcol_Firewall_Settings_for_IBM_SmartCloud_Engage_Advanced"><span style="color:#424282">public host names</span></a> </span>to IP addresses owned by its network points of presence worldwide (where
xx is the specific data center, EU = ce, US = na and Asia = ap): apps.xx.collabserv.com,
downloads.xx.collabserv.com, webchat.xx.collabserv.com<br>
This means that the IP addresses assigned to these domains will be dynamically
chosen from a long list of possible addresses and will no longer be a single
predictable address. This will allow for the best possible performance of the
public HCL Connections Cloud services.<o:p></o:p></p>
